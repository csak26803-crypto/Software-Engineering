# Software Engineering
# 券売所混雑緩和のための事前QRコード生成システム

## 1. 要件定義書

### 目的・利用者
* **目的:** 券売所に並ぶ列の緩和
* **利用者:** 券売所の利用者

### 入出力要件
* **入力:** チケットの数量やオプション（割引や企画チケット）
* **出力:** QRコード（受け取り媒体はモバイル決済なら携帯電話、券売所決済なら紙）

### 機能一覧（主要機能）
1.  **商品カタログ機能:** 店舗が提供するチケットの一覧、画像、価格、詳細を表示する。
2.  **カート機能:** 商品をカートに追加、確認、数量やオプションを変更、削除する。
3.  **商品データ集計機能:** 選択されたチケットID、数量、オプション情報から合計金額を計算し表示する。
4.  **QRコード作成・表示機能:** 確定したカート内のデータを、券売機側が読み取れるフォーマットに変換し、QRコードとして画面に描画・表示する。

### 受け入れ基準
* 商品一覧が正常に表示され、ユーザーが任意の複数商品とオプションを選択してカートに追加できること。
* カート内の合計金額が正しく計算され、商品の増減や削除が正常に行えること。
* 選択した商品データをもとにQRコードが即座に生成され、画面に表示されること。
* 生成されたQRコードを読み取りアプリ（または券売機エミュレータ）でスキャンした際、事前に定義された仕様通りに商品IDと数量・オプションのデータが欠損なく取得できること。

### 非目標（今回は実装しないこと）
* **オンライン決済システムとの連携:** 最終的な完全キャッシュレス化は目指すが、外部API連携・審査・セキュリティ実装に多大な工数がかかるため今回は除外。「注文の作成とQRコード化」に特化する。
* **実際の券売所ハードウェアとの直接連携:** 実機へのデータ流し込みはベンダー協力等が必要なため除外。PCやタブレット上のエミュレータ（確認用画面）でのスキャン検証にとどめる。
* **会員登録・顧客データ機能:** ポイント付与や購入履歴管理は将来の展望とし、今回は「混雑緩和」という第一課題の解決を優先するため認証基盤の構築は行わない。
